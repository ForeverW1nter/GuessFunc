# AI 核心指令与开发协议

> **致所有 AI 助手:** 
> **强烈警告：时刻记住所有指令，特别是本目录下的全局约束！**
> 本文档是最高指令。执行任何指令前，必须阅读并遵循此目录下的所有 `.md` 规范。

## 1. 强制同步与自检约束 (优先级最高)
1. **动手前显式自检 (Pre-flight Check)**：写代码前，必须在回复中输出 `[Pre-flight Check]` 小节，回答：
   - 架构：是否违反 FSD 边界？
   - 文本：是否有硬编码文本？（严禁硬编码，`i18n.t()` 严禁传 fallback）
   - 配置：是否有魔法数字/色值？（必须提取到 `constants.ts`）
   - 文档：是否已读 `hist.md` 避开雷区？
2. **完成前强制验证 (Post-flight Check)**：
   - 任务结束前，必须系统性验证（严禁凭直觉）。请使用命令验证：`npm run lint`, `npm run build` (tsc 类型检查), `npm run test`（若有 E2E，执行 `npx playwright test`）。
   - 必须通过全局搜索 (`Grep`或正则搜索) 检查代码库中是否包含未提取的魔法数字或硬编码文本。
   - **零容忍**：修复所有 Error 和 Warning，必须清理临时文件/脚本。
3. **知识沉淀**：任务结束后更新 `.trae/rules` (尤其是 `conn.md` 和 `hist.md`)，记录踩坑经验。

## 2. 核心开发哲学
- **拒绝盲猜**：Bug 排查必须严谨 (前置条件 -> 输入输出 -> 数据流 -> 隔离)。严禁凭直觉“盲猜”和临时“Hack”。
- **根除问题**：思考 Root Cause，通过全局搜索 (`SearchCodebase`/`Grep`) 找出所有牵连面。
- **严禁逃避性代码**：严禁使用 `@ts-ignore`, `any`, `// FIX`。使用严格类型推导或 Zod 类型守卫。
- **消除冗余与相关性检查**：无论什么时候，修改代码时都要检查与之相关的东西。相似的代码必须一起修改；如果逻辑特别相似，必须直接合并抽象，坚决避免冗余代码。这非常重要！
- **顺手重构与全局观**：如果看到不符合代码规范的地方请顺手改掉，但是**必须多思考并顾全大局**，在看完全貌后再进行修改，绝不能改完引入更多 Bug。

## 3. 编码与文档规范
- **严禁强制样式**：不能有任何形式的强制（如 `!important` 或 Tailwind 中的 `!前缀`）。必须通过调整选择器权重和结构来解决样式冲突。
- 单文件尽量不超过 300 行；复杂函数必须提取；命名必须语义化。
- **命名规范**：变量名、类名、配置项等名称**严禁**带有数字或具体的数值（如 `spin-10s`, `duration-300`, `margin-15` 等），必须使用纯粹描述功能、意图或状态的语义化单词（如 `spin-slow`, `duration-normal`, `margin-large` 等）。
- **原生控件限制**：严禁使用任何原生控件或提示框（如 `window.alert`, `window.confirm`, `window.prompt`），必须使用项目内置的自定义组件（如 `ConfirmDialog`, `Toast`）。
- **注释**：简体中文；导出接口用 JSDoc；行内用 `// `；**严禁**在注释中写推导/心理活动。
- **文档**：必须使用 **UTF-8**；**严禁**使用 Emoji；更新文档必须保持原有格式结构。
- **UI 规范**：界面严禁使用 Emoji，必须全部使用规定网站的图标。
