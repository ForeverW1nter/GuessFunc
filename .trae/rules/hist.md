## <Bug_Graveyard> 历史致命 Bug 警示录与防御策略

> 在修改相关模块时必须反复阅读，**绝对不允许**重蹈覆辙。

### 🚨 优先级 1：全站白屏或数据损毁
| 模块 | 症状 | 根本原因 (Root Cause) | 防御策略 (必须遵守) |
| :--- | :--- | :--- | :--- |
| **持久化 / JSON** | 导入非法存档导致白屏死循环。 | Zustand 还原前缺乏校验；`ErrorBoundary` 未清坏数据。 | **防御**：处理外部 JSON **必须**用 Zod；`ErrorBoundary` 重置需精准清缓存保核心进度。 |
| **Zustand Hydration** | 新用户首开抛 `expected object, received undefined`。 | 空 LocalStorage 被 `merge` 传为 `undefined` 导致 Zod 崩溃。 | **防御**：Zustand 自定义 `merge` **必须**首行拦截 `if (!persistedState) return currentState;` |
| **路由与状态机** | 错误 URL 或连点后退导致白屏挂起。 | `Loader` 未处理空索引；缺状态重置保护。 | **防御**：路由守卫获取参数失败**必须**重定向至安全路由，严禁透传空数据。 |
| **动态导入加载** | 点击故事模式提示找不到关卡并卡死。 | `import.meta.glob` 对象直接给 Zod 校验导致失败。 | **防御**：动态导入必须提取 `module.default || module`。验证报错必须告警/阻塞，严禁静默吞噬。 |
| **外部数据解析** | JSON 格式变动导致崩溃。 | 使用了不安全的 `as` 类型断言。 | **防御**：非受信数据**必须**使用 `Schema.parse()` 校验。 |
| **数学引擎编译** | 微积分节点导致 React 抛出异常白屏。 | `box.compile()` 未用 `try...catch` 包裹。 | **防御**：外部库编译/解析操作**必须** `try...catch` 兜底。 |
| **URL 解析安全** | Emoji 标题分享链接引发 `URIError`。 | 使用废弃的 `escape`/`unescape`。 | **防御**：**严禁**使用 `escape`。必须用 `btoa(encodeURIComponent(...))`。 |

### 🚨 优先级 2：性能或内存泄漏
| 模块 | 症状 | 根本原因 (Root Cause) | 防御策略 (必须遵守) |
| :--- | :--- | :--- | :--- |
| **第三方库性能** | 拖拽图表卡顿，切换关卡内存泄漏。 | `change` 事件同步解析 AST；`useEffect` 依赖滥用致频繁销毁。 | **防御**：高频事件**强制防抖**；非 React 实例依赖必须为 `[]`，内部用 `useStore.getState()`。 |
| **React 路由死循环**| 分享页切换导致浏览器卡死。 | 渲染期用 `useEffect` + `useNavigate` 重定向。 | **防御**：渲染期重定向**严禁**用 `useEffect`，必须用 `<Navigate replace />` 或 Router `loader`。 |
| **Hooks 内存泄漏** | 切走后报错“在已卸载组件上更新状态”。 | 定时器缺 `cleanup` 清理。 | **防御**：`useEffect` 中的全局事件或定时器**必须**返回清理函数。 |
| **微任务挤压** | 输入未完成表达式时响应迟缓。 | 异常捕获块滥用 `import()` 动态加载 Logger。 | **防御**：热路径内**严禁**动态导入非异步组件，必须顶部静态引入。 |

### 🚨 优先级 3：逻辑错误与扩展性
| 模块 | 症状 | 根本原因 (Root Cause) | 防御策略 (必须遵守) |
| :--- | :--- | :--- | :--- |
| **状态不同步** | 切空存档时画面残留上一关（幽灵状态）。| 仅重置进度未重置 `useGameStore` 状态机。 | **防御**：全局上下文切换时，**必须**同步重置所有相关 Store 状态。 |
| **硬编码依赖** | 增删函数导致无法获取玩家答案。 | 硬编码强依赖第三方库内部 `id === 'target-function'`。 | **防御**：**严禁**硬编码外部 ID。在 `constants.ts` 维护并在逻辑中用排除法识别。 |
| **数学引擎定义域** | 存在盲区的函数被错误判对。 | 循环匹配后未因 `domainMismatchCount` 否决。 | **防御**：采样循环严禁轻易 `continue`。必须联合判定有效点与定义域错误。 |
| **大型循环性能** | 判定失败后仍继续千次抖动采样。 | `evaluate2D` 循环体缺 `break`。 | **防御**：O(N) 级别大循环一旦确定失败**必须立即 break/return**。 |
| **极值与除零** | 等价验证极少数报错。 | 计算比例未防御除零并滥用 `as`。 | **防御**：除法必须做 `0` 校验；严禁粗暴类型断言。 |
| **渲染期状态突变** | 切关卡页面短暂闪烁。 | `Loader` 在渲染周期突变全局状态。 | **防御**：重定向和预加载必须在 Router `loader` 中进行。 |
| **硬编码翻译** | 切语言后错误信息仍为中文。 | 核心组件硬编码字符串。 | **防御**：面向用户文案**必须**用 `i18n.t()` 渲染。 |
| **章节合并去重失效** | 导入/批量生成后出现同 ID 的重复章节。 | 用 `id + title` 作为章节唯一键，语言切换或文案差异导致误判为不同章节。 | **防御**：章节合并唯一性**只能**依赖稳定标识 `id`；标题仅作展示字段，严禁参与去重判定。 |
| **全局样式污染** | 组件样式互相覆盖。 | `index.css` 堆砌自定义全局类。 | **防御**：严禁写全局自定义 CSS，必须用 Tailwind 或 React 组件封装。 |

### 历史重构里程碑 (严禁退化)
1. **类型与常量**：消除 `any`/`@ts-ignore`；提取硬编码至 `constants.ts`。
2. **架构解耦**：巨型组件肢解，复杂逻辑下放 Controller Hook；建立防腐层与导出白名单，修复 FSD 边界。
3. **性能极化**：耗时计算移至 Web Worker；长列表虚拟化；动画 GPU 加速；组件与第三方大库按需加载；预加载音频。
4. **安全防御**：Zustand 迁移强类型测试兜底；Zod 拦截脏数据；消除隐式断言。
5. **工程基建**：Playwright E2E/Worker 测试覆盖；i18next-parser 自动提取；jscpd 清除重复代码；完善未捕获异常监控；Lint/Test 清零报错。
6. **动态导入优化**：统一了 `mathEngine` 等模块的导入方式（全静态或全路由级按需懒加载），消灭了由于混合导入导致的 Vite 独立拆分 chunk 警告，进一步优化了产物体积。
7. **UI 组件基建沉淀 (Headless UI)**：引入了 Radix UI / shadcn/ui 无头组件库取代部分纯 Tailwind 配合自定义 className 的实现，增强了自定义组件的无障碍访问 (A11y) 和键盘交互能力。
8. **错误上报与监控闭环**：基于原有的 ErrorBoundary 和 SYSTEM_LOGS，无缝接入了 Sentry 前端监控 SDK，实现了 `logger.error` 堆栈自动上传和完整的错误监控闭环。
