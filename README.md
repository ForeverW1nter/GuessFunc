# GuessFunc (猜函数游戏)

这是一个基于现代前端技术栈 (React + TypeScript + Vite) 构建的数学解谜与函数可视化游戏。玩家需要通过观察给定的函数图像，利用自己掌握的数学知识（如多项式、三角函数、指数对数等），在输入框中构造出正确的函数表达式来拟合目标曲线。

## 核心特性

- **直观的函数可视化**：集成了强大的 Desmos 图形计算器引擎，支持复杂数学公式的实时渲染与交互。
- **智能的等价性验证**：不依赖死板的字符串匹配，而是通过抽象语法树 (AST) 解析与多点数值采样（蒙特卡洛方法），智能判断玩家输入的函数与目标函数在数学上是否真正等价（例如：`2x` 会被认为等价于 `x+x`）。
- **多种游戏模式**：
  - **故事模式**：跟随精心设计的关卡，逐步解锁更复杂的数学概念。
  - **随机挑战**：系统自动生成随机的高难度函数供玩家挑战。
  - **自由创作与分享**：玩家可以自己设计函数关卡，生成专属的分享链接发给朋友。
- **AI 辅助系统**：内置 AI 助手，可以通过提问来获取关于当前目标函数的数学特征提示。
- **完善的本地持久化**：游戏进度、解锁关卡与偏好设置均安全地保存在本地。

## 技术栈

- **前端框架**: React 18 + Vite
- **语言**: TypeScript (严格模式)
- **状态管理**: Zustand (结合 LocalStorage 持久化)
- **数学引擎**: 
  - 渲染：Desmos API
  - 解析验证：@cortex-js/compute-engine
- **路由**: React Router v6
- **样式**: Tailwind CSS + Lucide React (图标)
- **国际化**: i18next

## 快速开始

如果你想在本地运行或修改这个游戏：

### 1. 安装依赖
确保你已经安装了 Node.js (推荐 v20+)，然后在项目根目录下运行：

```bash
npm install
```

### 2. 启动开发服务器
直接双击根目录下的 `start.bat` 文件，或者在终端运行：

```bash
npm run dev
```

### 3. 构建与部署
项目已配置好 GitHub Actions 自动部署流程。如果要手动打包：

```bash
npm run build
```
打包产物会生成在 `dist` 目录中。

## 目录结构

项目采用了高内聚的 FSD (Feature-Sliced Design) 架构：
- `src/features/`：按业务功能划分的模块（如 `game`, `story`, `creation`, `ui`）。
- `src/store/`：Zustand 全局状态管理。
- `src/utils/`：工具函数，包括核心的数学验证引擎 (`mathEngine`) 和 Desmos 加载器。
- `src/locales/`：i18n 国际化文本配置。
