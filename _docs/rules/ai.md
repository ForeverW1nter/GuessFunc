# AI 核心指令

> **最高优先级：执行任何任务前必须遵循本目录下所有规范**

## 1. 强制检查流程

**Pre-flight Check（动手前）：**
- 架构：是否违反 FSD 边界？
- 文本：是否有硬编码文本？（`i18n.t()` 严禁传 fallback）
- 配置：是否有魔法数字/色值？（必须提取到 `constants.ts`）
- 文档：是否已读 `hist.md` 避开雷区？

**Post-flight Check（完成前）：**
- `npm run lint` - 零 Error/Warning
- `npm run build` - 类型检查通过
- `npm run test` - 所有测试通过
- 全局搜索魔法数字和硬编码文本
- 清理临时文件和脚本

## 2. 核心原则

- **拒绝盲猜**：Bug 排查必须严谨（前置条件→输入输出→数据流→隔离）
- **根除问题**：通过全局搜索找出所有牵连面
- **严禁逃避性代码**：禁用 `@ts-ignore`, `any`, `// FIX`
- **消除冗余**：相似代码必须合并抽象，一起修改
- **全局观**：顺手改规范，但必须看完全貌后再修改

## 3. 关键约束

- **禁止强制样式**：不用 `!important` 或 `!` 前缀，通过选择器权重解决
- **命名严禁数字**：用语义化单词（`spin-slow` 而非 `spin-10s`）
- **禁止原生控件**：不用 `window.alert`，用项目内的 `ConfirmDialog`, `Toast`
- **注释规范**：简体中文，JSDoc 写接口，行内用 `// `，不写推导过程
- **文档格式**：UTF-8，禁用 Emoji，保持原结构
- **UI 规范**：界面禁用 Emoji，用图标库

## 4. 规则文档索引

- `ai.md` - 核心指令（本文档）
- `hist.md` - 历史 Bug 防御策略（必读）
- `proj.md` - 编码与架构规范
- `map.md` - 目录结构
- `tasks.md` - 待办任务
- `conn.md` - 近期任务上下文
