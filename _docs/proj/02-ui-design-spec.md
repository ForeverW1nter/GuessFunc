# 沉浸式微内核游戏平台 UI 设计方案

本方案旨在为微内核游戏平台设计一套**多端适配**、**沉浸体验**、**极致平滑**、且**高度可定制**的高级感 UI。我们抛弃传统的“后台管理面板”风格（侧边栏+顶部栏+大白卡片），采用**“全屏沉浸式操作系统” (Immersive OS-like Interface)** 的设计语言。

---

## 1. 核心设计理念 (Core Philosophy)

*   **沉浸式体验 (Immersive)**：没有冗余的边框和生硬的卡片。UI 元素似乎是“悬浮”在背景之上的。进入游戏时，平台 UI 会优雅地淡出或隐藏，绝不抢占游戏的视觉焦点。
*   **直觉化操作 (Intuitive)**：扁平化层级。用户永远知道自己在哪，该怎么去别的功能。
*   **绝对平滑 (Silky Smooth)**：拒绝闪烁和生硬切换。采用基于弹簧物理 (Spring Physics) 的动画引擎（如 Framer Motion 或原生 CSS `transition: all 0.4s cubic-bezier(...)`），所有的进入、退出、Hover 都如同水流般顺滑。
*   **去 AI 味/去原生化 (Custom & Premium)**：
    *   绝不使用原生 `<select>`, `<input type="range">`, 原生滚动条。
    *   抛弃烂大街的 Inter / Roboto 字体，主标题使用极具几何感或衬线的高级字体（如类似 *Clash Display* 或 *Playfair Display* 风格的无免费版权替代），正文使用极其干净的无衬线体。
    *   告别刺眼的白色背景，浅色模式采用极淡的暖灰/米色纸张感，深色模式采用深邃的深渊黑/碳灰。

---

## 2. 交互流与功能地图 (User Flow & Routing)

为了实现“找功能不费劲”，我们将所有功能收敛为 **3 个核心层级**：

### 层级 A：主控制台 (The Hub - `/`)
**功能**：入口、游戏列表、全局状态一览。
**布局**：
*   **桌面端**：**全屏画廊 (Gallery View)** 或 **大视差网格 (Bento Grid)**。没有侧边栏。屏幕中央是巨大的、有电影感的游戏海报/卡片。
*   **移动端**：纵向的大卡片流 (Feed View)，利用手机的惯性滚动，滑动时卡片带有轻微的缩放（Scale）和视差（Parallax）效果。
*   **全局导航台 (Command Bar)**：屏幕底部或顶部悬浮的一个胶囊状的毛玻璃导航条（Dock）。包含：`[主页] [创意工坊] [设置] [个人成就]`。这个 Dock 在所有非游戏页面常驻，进入游戏后自动隐藏为一个小小的悬浮按钮。

### 层级 B：全局面板 (Overlays)
**功能**：设置、主题、存档管理、通知。
**布局**：
*   **非跳页设计**：设置和模组商店等不需要离开当前页面。它们通过**全屏毛玻璃遮罩 (Full-screen Glass Overlay)** 或 **优雅的侧边抽屉 (Fluid Drawer)** 滑出。
*   **设置中心**：
    *   **高度自定义**：提供一个“色盘”和“主题选择器”。用户可以自定义平台的重音色（Accent Color）。
    *   提供“深色”、“浅色”以及“纯黑（OLED 友好）”选项。

### 层级 C：沉浸游戏区 (The Game Zone - `/guessfunc` 等)
**功能**：具体的游戏游玩。
**布局**：
*   **全屏接管**：点击主页游戏卡片后，卡片通过 **Shared Element Transition（共享元素动画）** 放大占据全屏，平滑过渡到游戏界面。
*   **悬浮退出键**：屏幕角落只有一个极简的悬浮“×”或“返回”按钮（带毛玻璃背景），鼠标靠近时才会完全显现。
*   **游戏内工具栏**：游戏特有功能（如难度滑块、重置）收纳在一个可通过手势/快捷键唤出的浮动面板中，绝不占用游戏画面的常驻空间。

## 3. 视觉与组件规范 (Visual & Component Specs)

### 3.1 色彩与主题 (Color & Theming)
基于 CSS Variables 构建，支持无限扩充主题。
*   **背景 (Background)**：带有极微弱的噪点材质（Noise Texture），增加质感。
*   **层级结构**：不再用厚重的 Box Shadow 来区分层级，而是利用背景色的微小亮度差异和 `1px` 极细的环境光边框（Inner Glow）。
*   **重音色 (Accent Color)**：极简环境中的那一抹亮色。用于聚焦的按钮、进度条、激活的 Tab。默认使用高级的霓虹色（如 Neon Cyan 或 Vibrant Orange），并开放给用户自定义。

### 3.2 统一的控件库 (Unified Headless UI)
**严禁使用任何原生控件。所有控件风格高度统一。**

*   **按钮 (Buttons)**：没有死板的矩形块。采用大圆角药丸状 (Pill-shaped) 或纯文字带极细下划线（Underline reveal on hover）。点击时有类似于液体波纹的点击反馈或明显的缩放反馈 (`scale: 0.95`)。
*   **下拉菜单 (Select/Dropdown)**：点击后，选项列表以毛玻璃卡片的形式从中心向外展开（Scale & Fade），选项之间没有分割线，只通过 Hover 背景色区分。
*   **滑块 (Slider/Range)**：极细的轨道（Track），加上发光的滑块柄（Thumb）。拖动时上方浮现一个精致的气泡显示当前数值。
*   **模态框 (Modals)**：摒弃居中的死板白框。采用占据屏幕大半的毛玻璃浮层，或者底部抽屉（移动端完美适配，可下拉关闭）。

### 3.3 动画与微交互 (Motion & Micro-interactions)
*   **Staggered Fade-in（交错淡入）**：页面加载时，元素不要同时闪出。标题、副标题、卡片列表应该按照 `0.1s` 的间隔，从下往上优雅地浮现（`y: 20, opacity: 0` -> `y: 0, opacity: 1`）。
*   **Magnetic Hover（磁性吸附）**：桌面端，当鼠标靠近某些关键按钮（如悬浮返回键）时，按钮会稍微向鼠标方向偏移，产生“被吸住”的物理反馈。
*   **Skeleton/Loading**：告别转圈圈的 Spinner。使用带有柔和渐变流光的骨架屏，或高级感的呼吸灯效果。

## 4. 技术实现路径 (Implementation Strategy)

为了实现上述效果，在我们的微内核架构下，实施步骤如下：

1.  **引入动画引擎与 Headless UI**：
    *   继续沿用 TailwindCSS（或配置为纯 CSS 变量体系）处理基础布局。
    *   引入 `framer-motion` 处理共享元素动画（Shared Layout Animations）和弹簧物理动画。
    *   引入 `@radix-ui/react-primitives` 或 `ariakit` 作为无样式底层组件，彻底接管下拉、滑块、弹窗的无障碍和焦点管理（完全替代原生控件），然后在上面套上我们的高级样式。
2.  **重构 UI Manager (`mod-ui-manager`)**：
    *   将我们在架构中设计的 `UIProvider` 扩充。它将不仅管理主题，还要管理全局抽屉（Drawer）、全局命令面板（Command Bar）、以及自定义鼠标光标（可选的高级感细节）。
3.  **构建 Command Bar (Dock)**：
    *   实现全局悬浮导航栏，处理在移动端和桌面端的形态切换。
4.  **重构主页 (The Hub)**：
    *   应用大卡片/画廊设计，实现交错淡入和 Hover 视差。
5.  **设计游戏内壳 (Game Shell)**：
    *   设计那个悬浮的、极简的退出/设置控制台，确保当 GuessFunc 等插件注入时，它们运行在一个完美的、不受干扰的画布上。

---

## 5. 总结：最终效果预期

当用户打开平台：
*   迎面而来的是干净、高级的纸张感（或深渊感）背景，伴随轻微噪点。
*   大标题和游戏大卡片如丝般顺滑地依次浮现。
*   鼠标移动到游戏上，背景光晕轻微变化，没有死板的边框变化。
*   点击设置，全屏毛玻璃柔和覆盖，一个色盘优雅展开，用户点选一抹霓虹蓝，整个网站的焦点色瞬间顺滑过渡。
*   点击“GuessFunc”，卡片瞬间放大接管全屏，进入沉浸的解谜世界，周围没有任何多余的导航栏干扰。

这就是我们要追求的“去 AI 味”的极致前端美学方案。
