# GuessFunc 剧情与关卡配置指南

本文档将详细介绍如何在**不修改任何核心代码**的情况下，通过修改配置文件来添加、修改线路剧情，以及设置关卡和章节的解锁条件。

---

## 1. 核心配置文件：`js/levels.js`

所有的游戏线路、关卡、章节、解锁条件和剧情文本路径都在 `js/levels.js` 文件中通过 JSON 结构进行配置。游戏会自动读取该文件并动态生成 UI。

> **注意：** 该文件是一个 `.js` 文件，所以配置内容需要包裹在 `const ROUTES = /*===ROUTES_START===*/` 和 `/*===ROUTES_END===*/;` 之间，以保证前端能够直接加载。

### 1.1 默认线路的决定机制
游戏在启动时，会**默认加载 `ROUTES` 数组中的第一个线路（索引为 0 的线路）**。
- 如果你想让某条线路成为默认线路（比如 `AI2`），只需要将该线路的 JSON 对象移动到数组的最前面即可。

---

## 2. 添加或修改线路

要添加一条新线路，只需要在 `ROUTES` 数组中添加一个新的对象。新添加的线路会自动在游戏顶部的 Tab 栏生成一个新的切换按钮。

### 线路对象结构示例：

```javascript
{
    "id": "new_route",            // 线路的唯一标识符（存档隔离依赖此ID，必须全局唯一）
    "title": "新线路的名称",        // 选关界面顶部 Tab 显示的名称
    "description": "线路的一句话简介", 
    "endingPath": "story/endings/new_route_ending.md", // (可选) 通关该线路最后一关后解锁的真结局剧情路径
    "levels": [ ... ],            // 该线路包含的所有关卡数组
    "regions": [ ... ]            // 该线路的章节划分和章节剧情
}
```

---

## 3. 配置关卡 (`levels` 数组)

`levels` 数组包含了该线路下所有的关卡。所有的关卡必须在这里注册。

### 关卡对象结构：

```javascript
{
    "id": 1,                      // 关卡的唯一标识符（数字或字符串皆可，如 "new_1"）
    "title": "第 1 关",            // 关卡显示的名称
    "target": "x+1",              // 需要猜出的目标函数 (支持 LaTeX，如 "\\frac{1}{x}")
    "descriptionPath": "story/new_route/ch0/level1.md", // (可选) 该关卡的剧情 Markdown 文件路径
    "unlock": { ... }             // (可选) 关卡的独立解锁条件，见第5节
}
```

---

## 4. 配置章节与假结局 (`regions` 数组)

`regions` 用于将关卡分组，并提供章节级的剧情和“假结局”。游戏会自动按此结构生成带有折叠和锁定提示的章节面板。

### 章节对象结构：

```javascript
{
    "id": "new_route_ch0",
    "title": "序章：启程",
    "descriptionPath": "story/new_route/ch0/story.md", // (可选) 章节介绍剧情，看过一次后UI会显示"剧情"按钮
    "levels": [ 1, 2, 3 ],        // 属于该章节的关卡 ID 列表（必须与 levels 中定义的 ID 对应）
    "unlock": { ... },            // (可选) 整个章节的解锁条件，见第5节
    "fakeEndings": [              // (可选) 假结局数组，可以配置多个
        {
            "id": "new_route_fake1",
            "title": "中途放弃的结局", // 按钮上显示的名称
            "descriptionPath": "story/endings/fake_ending1.md",
            "unlock": {           // 假结局的触发条件
                "type": "specific_levels",
                "target": [1, 2]  // 当完成第1、2关时触发
            }
        }
    ]
}
```

### 关于假结局的触发与回顾机制：
- **触发**：玩家在闯关模式下通关某个关卡后，系统会自动检查当前章节下的 `fakeEndings`。如果某个假结局的 `unlock` 条件被满足，并且玩家**之前从未看过**，游戏会立即中断正常的“下一关”流程，弹出该假结局剧情。
- **回顾**：只要假结局被触发（或通过存档/修改器标记为已看过），章节标题栏的右侧就会永久显示一个紫色的按钮（如“中途放弃的结局”）供玩家随时点击回顾。不需要再次满足条件。

---

## 5. 解锁条件配置 (`unlock` 属性)

如果不配置 `unlock`，游戏会使用默认规则：**上一章节完成度达到80%解锁下一章；本章最多允许同时存在3个未通关关卡**。
如果配置了 `unlock`，则完全由数据驱动。支持配置在 `levels`（单关卡解锁）、`regions`（整章解锁）或 `fakeEndings`（假结局触发）中。

你可以传入一个**条件对象**，或者一个**包含多个条件对象的数组**（必须全部满足）。

### 支持的解锁类型 (`type`)：

1. **指定关卡完成 (`specific_levels`)**
   需要玩家完成指定的关卡 ID 才能解锁。
   ```javascript
   "unlock": {
       "type": "specific_levels",
       "target": [ 1, 2 ] // 必须完成 ID 为 1 和 2 的关卡
   }
   ```

2. **总关卡数要求 (`total_completed`)**
   玩家在当前线路中总共完成的关卡数量达到要求。
   ```javascript
   "unlock": {
       "type": "total_completed",
       "target": 10 // 总共需要通关至少 10 关
   }
   ```

3. **指定章节完成度 (`chapter_percentage`)**
   需要某个特定章节完成指定的百分比。
   ```javascript
   "unlock": {
       "type": "chapter_percentage",
       "target": {
           "chapterId": "AI2_ch0",
           "percentage": 0.5 // 需要 AI2_ch0 章节完成 50%
       }
   }
   ```

4. **指定章节通关数量 (`chapter_completed_count`)**
   需要某个特定章节完成指定数量的关卡。
   ```javascript
   "unlock": {
       "type": "chapter_completed_count",
       "target": {
           "chapterId": "AI2_ch0",
           "count": 5 // 需要 AI2_ch0 章节完成 5 关
       }
   }
   ```

## 7. 支持的函数及格式参考

在编写 `formulas_input.txt` 时，请参考以下支持的数学函数及其推荐的 LaTeX 书写格式。我们的引擎（以及后端的 Desmos 渲染）对这些格式有良好的兼容性：

### 基础运算
- **加法**: `x + 1`
- **减法**: `x - 1`
- **乘法**: 可以直接连写 `2x`, `ax`, 或者使用 `\cdot`：`2 \cdot x` 或者用星号 `2*x`（生成器会自动转换为 `\cdot`）
- **除法 / 分数**: `\frac{1}{x}` (推荐) 或 `1/x`
- **指数 / 幂**: `x^2`, `x^{3+a}` (指数部分复杂时务必用 `{}` 括起来)

### 常用函数 (必须带反斜杠 `\`)
- **绝对值**: `\left| x \right|` (推荐)、直接用竖线 `|x|`，或者使用 **`\abs(x)`** / **`\abs{x}`** (引擎兼容这两种写法)。
- **自然对数**: `\ln(x)`
- **以10为底的对数**: `\log(x)`
- **平方根**: `\sqrt{x}`
- **n次方根**: `\sqrt[3]{x}` (表示x的三次方根)
- **圆周率 Pi**: `\pi` (注意是小写且必须带反斜杠)

### 指数函数 (特别注意)
- **自然指数**: 推荐直接写为 **`e^x`** 或 **`e^{2x}`**。
  - *(注：引擎兼容解析 `\exp(x)`，但为了在 Desmos 中呈现最自然的输入体验，强烈推荐使用 `e^{...}` 的格式)*。

### 三角函数与反三角函数
- **正弦**: `\sin(x)`
- **余弦**: `\cos(x)`
- **正切**: `\tan(x)`
- **反正弦**: `\arcsin(x)` (注意：不要写成 `\asin`)
- **反余弦**: `\arccos(x)` (注意：不要写成 `\acos`)
- **反正切**: `\arctan(x)` (注意：不要写成 `\atan`)

### 双曲函数
- **双曲正弦**: `\sinh(x)`
- **双曲余弦**: `\cosh(x)`
- **双曲正切**: `\tanh(x)`

> **💡 小贴士**：所有的内置函数名（如 sin, cos, ln, sqrt 等）前面都**必须**加上反斜杠 `\`。这是标准的 LaTeX 语法，能确保图形引擎正确地将它们识别为函数，而不是变量相乘（例如 `sin` 会被识别为 `s*i*n`）。

辅助脚本位于 `scripts/` 目录下，用于快速生成或批量修改数据。
**执行前警告**：脚本会通过正则表达式直接修改 `js/levels.js`，建议在执行前确保代码已提交至 Git，或备份 `levels.js`。

### 6.1 剧情生成脚本 (`generate_story.js`)
**功能**：
- 自动在 `curr/story/` 下创建对应的多级目录（如 `story/AI1/ch0/` 和 `endings/`）。
- 自动生成对应的 `.md` 文本文件并填入内容。
- 自动解析文本结构，并更新 `js/levels.js` 中的 `regions`、`levels` 以及 `fakeEndings` 结构。

**使用方法**：
1. 在 `scripts` 目录下创建 `story_input.txt`。
2. 按以下规范编写文本：
   ```text
   @ROUTE: LINEID               # 指定要更新的线路 ID
   @TITLE: 线路名称
   @DESC: 线路简介
   
   # 章节名称1               # 生成新的 Region
   这是第一章的总剧情。                # 填入 story.md 的内容
   
   ## 关卡剧情               # 识别为该章节下的一个新关卡，生成 level1.md
   这是第1关的内容。
   
   ## 假结局：中途放弃的结局 | 2   # 识别为一个假结局（数字2表示完成该章前2关触发，不写则默认完成该章所有关卡后触发）
   这是假结局的内容。玩家在打完这章第2关后会触发这个剧情。

   ## 真结局                 # 必须写在最后，整个线路打通后播放
   这是整个游戏的最终结局。
   ```
3. 在终端运行：`node scripts/generate_story.js`

### 6.2 公式填充脚本 (`fill_formulas.js`)
**功能**：
- 批量替换 `levels.js` 中某条线路的 `target` 字段和 `params` 字段。
- 自动对纯文本公式进行简单的 LaTeX 兼容处理。
- **支持带参数关卡**：如果你想让用户猜测带有未知参数的表达式（例如 `a*x+b`），你可以在公式后面加上 `| 参数名=初始值`，生成器会自动把这些参数传递给游戏逻辑。用户在游戏中滑动这些参数滑块，最终必须猜出带有字母 `a` 和 `b` 的表达式才算通关。

**使用方法**：
1. 在 `scripts` 目录下创建 `formulas_input.txt`。
2. 第一行指定 `@ROUTE: 线路ID`，从第二行开始每行写一个公式。公式的顺序将严格对应该线路 `levels` 数组中的关卡顺序（第1行对应第1关，以此类推）。
3. 如果需要为关卡指定可调节参数，在公式后加上 `|` 符号，然后以 `参数名=初始值` 的形式写在后面，多个参数用逗号分隔。

**格式示例**：
```text
@ROUTE: AI1
x+1
2x
\sin(x)
a*x^2 + b | a=1, b=0
a\sin(b*x) | a=2, b=1
\frac{1}{x}
```
4. 在终端运行：`node scripts/fill_formulas.js`
