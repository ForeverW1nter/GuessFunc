我正在制作一款名为《GuessFunc》的解密游戏。剧情通过一系列的“线路 (Routes)”来呈现，线路包含“章节 (Chapters)”，章节中包含“关卡 (Levels)”（我已自行设计好）和“文件 (Files)”（你来负责生成的剧情文本，如日志、聊天记录、备忘录等）。

我已经构建好了线路、章节和关卡的 JSON 框架，**请你不要修改或生成任何新的关卡数据（Levels），也不要改变原有的线路和章节设置。你只需要在每个章节的 `files` 数组中，填充引人入胜的剧情文件即可。注意文笔要自然，更像人写的而不是 AI 写的，章节标题也要像正常小说的章节标题。**

### 1. 剧情文件数据结构

你只需在原 JSON 的 `files` 数组中插入对象，对象必须严格遵守以下 TypeScript 接口定义：

```typescript
export interface FileData {
  id: string; // 确保文件 id 唯一（例如：ch1_file1）
  title: string; // 文件标题
  extension: string; // 文件扩展名，决定了文本在游戏中的排版与渲染方式（详见第2点）
  content: string; // 实际的剧情内容文本
  unlockConditions: string[]; // 解锁该文件所需完成的 level id。例如 ["1", "2"]，如果默认解锁请留空数组 []
}
```

### 2. 文件扩展名与格式限制（非常重要）

不同的 `extension` 决定了游戏内的渲染器，部分格式**不支持** Markdown 语法：

- **支持 Markdown 的类型**：
  - `md`：标准的 Markdown 文档。支持粗体、斜体、引用 `>`、代码块 ` ``` ` 等。
  - `note` / `memo`：私人笔记或备忘录。支持基础 Markdown。
- **纯文本/等宽字符渲染（不支持 Markdown，请用纯文本和换行排版）**：
  - `txt`：纯文本文件。
  - `log`：系统或终端日志。每一行最好有时间戳前缀，如 `[2026-04-05 14:02:11] SYS.WARN`。
  - `msg` / `trans`：截获的聊天记录或通讯录音转录。请使用纯文本排版对话，例如：
    ```
    Alice: 听得到吗？
    Bob: 信号很差，长话短说。
    ```
  - `cpp` / `dat` / `letter` 等自定义后缀：默认均按等宽纯文本渲染，适合展示伪代码或原始数据。

### 3. 约束与格式要求

1. **节奏与解锁**：请将文件分散在玩家游玩的过程中。例如：章节开头放一个默认解锁的 `[]` 引导剧情，第一关打完解锁一个 `.log` `["1"]`，第五关打完解锁最终对话 `.trans` `["5"]`。
2. **语言**：请使用**简体中文**编写故事内容和标题。
3. **仅输出 JSON**：你的完整回复必须是一个单一的、合法的 JSON 块。请勿包含任何介绍或总结性文本，直接输出我提供给你的结构，并把 `files` 数组填满即可。
